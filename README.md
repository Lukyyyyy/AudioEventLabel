# 音频事件标注工具 (Audio Event Annotation Tool)

这是一个基于 Python (PyQt5 + PyQtGraph) 开发的轻量级音频事件检测（SED）数据标注工具。允许用户可视化音频波形，标注起止时间点，并为音频事件分配类别标签。

![示例图片](app.png)

## ✨ 主要功能

- **波形可视化**：支持多种音频格式（wav, mp3, flac 等）的波形加载与显示。

- **灵活标注**：通过双滑块精准控制事件的"开始时间"和"结束时间"。

- **播放控制**：支持全曲播放和仅播放选中区域（方便确认标注准确性）。

- **任务管理**：

  - 自动将文件分为"未处理"和"已处理"列表。
  
  - 保存标注后，文件自动移动至"已处理"列表。
  
  - 支持断点续标，点击"已处理"文件可回显并修改标注。

- **配置持久化**：

  - 自动记录用户的标注保存路径。
  
  - 自动记忆上次使用的类别定义文件。
  
  - 通过 `config.json` 保存配置，支持桌面快捷方式启动。

## 🛠️ 环境依赖

### 1. 系统要求

- Python 3.8+
- 操作系统：Windows / macOS / Linux

### 2. 安装 Python 库

请确保安装了以下依赖库：

pip install PyQt5 pyqtgraph pydub sounddevice numpy

> 注意：`pydub` 依赖 ffmpeg 来处理非 wav 格式（如 mp3）。
>
> - Windows: 下载 ffmpeg 并将 bin 目录添加到系统环境变量 Path 中。
> - Mac: `brew install ffmpeg`
> - Linux: `sudo apt install ffmpeg`

## 🚀 快速开始

### 第一步：准备类别文件

创建一个 JSON 文件（例如 `categories.json`），在其中定义你需要的事件类别。格式必须为字符串列表：

[
    "Speech",
    "Dog_Bark",
    "Glass_Shatter",
    "Gunshot",
    "Scream"
]

### 第二步：启动程序

运行主程序文件：

python app.py

### 第三步：初始化配置

首次运行时，程序会检查配置文件。如果未检测到类别文件，会提示你选择：

- 在弹出的对话框中，选择你在第一步创建的 JSON 文件。

程序会自动创建默认的标注保存目录（默认为程序同级目录下的 `annotations` 文件夹）。

### 第四步：开始标注

1. **导入音频**：点击菜单栏 `文件 -> 打开文件` 或 `打开文件夹`。
2. **选择区域**：拖动下方的"开始"和"结束"滑块选择事件范围。
3. **确认类别**：在右侧下拉框选择对应的事件类别。
4. **添加记录**：点击 `添加标注` 按钮，波形图上会显示对应的色块。
5. **保存**：点击 `保存结果`。对应的 JSON 文件将生成，且当前音频会移动到"已处理"列表。

## ⚙️ 设置与配置

在菜单栏的 `"设置"` 中，你可以随时修改以下选项：

- **设置标注保存路径**：更改 JSON 标注文件的输出文件夹。修改后，程序会自动刷新左侧的文件列表状态（重新判断哪些已处理）。

- **导入类别文件**：加载新的事件类别列表。

### 关于 config.json：

程序根目录下会自动生成 `config.json`，用于存储你的偏好设置。请勿手动破坏其 JSON 结构。

{
    "annotation_dir": "D:/Research/Audio_Data/Annotations",
    "categories_path": "D:/Research/Audio_Data/classes.json"
}

## 📂 输出数据格式

每个音频文件会对应生成一个同名的 `.json` 文件。

示例： 如果音频文件名为 `record_001.wav`，生成的标注文件 `record_001.json` 内容如下：

[
    {
        "start": 1.520,
        "end": 3.850,
        "category": "Dog_Bark"
    },
    {
        "start": 5.100,
        "end": 6.200,
        "category": "Scream"
    }
]

- `start`: 事件开始时间（秒）。
- `end`: 事件结束时间（秒）。
- `category`: 事件类别。

## ⌨️ 快捷操作与提示

- **删除标注**：在右侧"音频事件"列表中，右键点击某条记录即可删除。

- **删除文件**：在左侧文件列表中，右键点击文件可将其从列表中移除（如果是"已处理"列表，还会询问是否同时删除对应的 JSON 文件）。

- **滑块联动**：拖动开始滑块超过结束滑块时，两者会自动推挤，防止时间逻辑错误。

## 📝 常见问题 (FAQ)

**Q: 为什么加载 mp3 文件报错？**
A: 请检查是否安装了 ffmpeg 并配置了环境变量。`pydub` 库处理压缩音频需要它。

**Q: 手动修改了 config.json 导致程序打不开怎么办？**
A: 直接删除 `config.json` 文件，重启程序，它会自动恢复默认设置。

**Q: 可以同时标注多个重叠的事件吗？**
A: 可以。你可以多次拖动滑块并点击"添加标注"，不同事件在时间轴上可以重叠。
